2019/12/29 카테고리 web

type script에 대해 알아보자 (4) - interface

Interface

  • 일반적으로 타입체크를 위해 사용되며 변수, 함수, 클래스에 사용할 수 있다.
  • ES6 에서는 인터페이스를 지원하지 않지만 TypeScript 에서는 지원한다.
  • 여타 다른 정적타입 언어에서 많이 쓰이는 인터페이스와 유사하게 사용된다.
  • 어떠한 객체가 가지는 프로퍼티, 메소드를 선언하는 것이다.

사용법

지금까지 사용했던 것처럼 함수가 전달받는 인자의 타입을 정의해줄 수 있었다.

labelledObj는 string 타입의 label 이라는 프로퍼티를 가져야 한다고 정의하고 있다.

위에 코드를 interface 를 사용하도록 다시 작성해보자

Optional Properties

인터페이스에서 정의한 모든 프로퍼티가 항상 필요하지 않을수도 있는데 이때는 ? 키워드를 이용하면 된다.

SqueareConfig 인퍼테이스는 color, width 프로퍼티가 있어도되고 없어도 된다고 명시하고 있다.

이를 구현하는 createSquare 함수에서는 각 프로퍼티가 있을때, 없을때를 구분하여 구현해주면 된다.

Readonly Properties

인터페이스를 정의할 때 readonly 키워드를 이용해 수정불가능 하도록 강제할 수도 있다.

변수를 const로 정의한 것과 같은 효과를 볼 수 있다.

불변한 값을 만들고 싶을 때 변수는 const 를 사용하고 프로퍼티는 readonly 를 사용하는 정도의 차이가 있다.

함수

함수에 인터페이스를 적용할 수 있다.

인터페이스에서 함수가 가져야하는 인수의 타입, 반환값의 타입을 지정한다.
