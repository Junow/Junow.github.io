TypeScript-2

2019/12/29 카테고리 web

type script에 대해 알아보자 (2) - 개요

TS 의 특징

  • typescript는 결국 javascript + type인 것이다.
  • javascript는 자료형에 대한 지정이 없다. let, const 등을 이용해서 선언만 할뿐.
  • javascript 코드를 실행하기 전에 tsc를 이용한 트랜스파일링 과정을 통해 안정성을 확보할 수 있다.
  • 타입을 미리 알수 있기 때문에 다음과 같은 예외처리가 필요없어진다.

인터페이스 (Interfaces)

이전 글 에서 간단하게 작성했던 코드에 인터페이스를 지정해보자

인터페이스는 객체가 가져야하는 형태를 지정해주는 역할이다.

여기서는 Person 객체는 string 타입의 firstName 과 lastName 을 가져야 한다고 명시하고 있다.

빌드하고 실행해보자

클래스 (Classes)

javascript에서 클래를 사용할때도 typescript를 이용할 수 있다.

클래스의 생성자에서 받는 인자들의 타입을 모두 string으로 지정한다.

이렇게 하면 firstName, middleInitial, lastName 을 이어붙여 만든 Student 클래스의 fullName은 항상 string 타입이라는 안정성을 얻을 수 있다.

어김없이 빌드후 실행

컴파일 할 때 옵션을 줄 수 있는데 --watch 옵션을 줘서 ts 파일이 바뀔때마다 빌드를 할 수 있다.

여기서 ts 파일을 수정하면 변화를 감지하고 다시 컴파일 한다. (매우빠름)
